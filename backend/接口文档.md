# 知识库服务接口文档

**Base URL**: `http://localhost:8000`  
**认证**: 除健康检查外，所有接口需在请求头中携带 JWT：`Authorization: Bearer <access_token>`。

快速开始（获取 token）：

1) 注册（需邀请码）
```bash
curl -s -X POST http://localhost:8000/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{"invite_token":"YOUR_INVITE_TOKEN","username":"alice","password":"password123"}'
```

2) 登录
```bash
curl -s -X POST http://localhost:8000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"alice","password":"password123"}'
```

响应会返回 `access_token`，后续请求示例统一用：
```bash
-H "Authorization: Bearer $TOKEN"
```

## 2. 知识库

### 2.1 创建知识库

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/knowledge-bases` | 创建知识库 |

**请求体**:
```json
{
  "name": "我的知识库",
  "description": "可选描述"
}
```

```bash
curl -s -X POST http://localhost:8000/api/knowledge-bases \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"name":"我的知识库","description":"测试用"}'
```

**响应字段**: `id`, `user_id`, `name`, `description`, `current_version_id`, `created_at`

---

### 2.2 列表知识库

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/knowledge-bases` | 当前用户知识库列表 |

```bash
curl -s http://localhost:8000/api/knowledge-bases \
  -H "Authorization: Bearer $TOKEN"
```

---

### 2.3 获取知识库

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/knowledge-bases/{kb_id}` | 获取单个知识库 |

```bash
curl -s http://localhost:8000/api/knowledge-bases/1 \
  -H "Authorization: Bearer $TOKEN"
```

---

### 2.4 更新知识库

| 方法 | 路径 | 说明 |
|------|------|------|
| PATCH | `/api/knowledge-bases/{kb_id}` | 更新名称/描述（可选字段） |

**请求体**:
```json
{
  "name": "新名称",
  "description": "新描述"
}
```

```bash
curl -s -X PATCH http://localhost:8000/api/knowledge-bases/1 \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"name":"更新后的名称"}'
```

---

### 2.5 删除知识库

| 方法 | 路径 | 说明 |
|------|------|------|
| DELETE | `/api/knowledge-bases/{kb_id}` | 删除知识库及其向量数据 |

```bash
curl -s -o /dev/null -w "%{http_code}" -X DELETE http://localhost:8000/api/knowledge-bases/1 \
  -H "Authorization: Bearer $TOKEN"
```

**成功**: HTTP 204，无响应体。

## 3. 文档与检索

### 3.1 上传 PDF

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/knowledge-bases/{kb_id}/documents/upload-pdf` | 上传 PDF，解析并写入向量库 |

**Content-Type**: `multipart/form-data`，字段名 `file`。

```bash
curl -s -X POST http://localhost:8000/api/knowledge-bases/1/documents/upload-pdf \
  -H "Authorization: Bearer $TOKEN" \
  -F "file=@/path/to/document.pdf"
```

**响应示例**: `{"kb_id":1,"source_id":"pdf_xxx","source_type":"pdf","chunks_count":5}`

---

### 3.2 录入网址

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/knowledge-bases/{kb_id}/documents/upload-url` | 根据 URL 抓取网页正文并写入向量库 |

**Content-Type**: `application/x-www-form-urlencoded` 或 `multipart/form-data`，字段名 `url`。

```bash
curl -s -X POST http://localhost:8000/api/knowledge-bases/1/documents/upload-url \
  -H "Authorization: Bearer $TOKEN" \
  -d "url=https://example.com/page"
```

**响应示例**: `{"kb_id":1,"source_id":"url_xxx","source_type":"url","chunks_count":3}`

---

### 3.3 上传纯文本

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/knowledge-bases/{kb_id}/documents/upload-text` | 提交纯文本并写入向量库 |

**Content-Type**: `application/x-www-form-urlencoded` 或 `multipart/form-data`，字段名 `text`。

```bash
curl -s -X POST http://localhost:8000/api/knowledge-bases/1/documents/upload-text \
  -H "Authorization: Bearer $TOKEN" \
  -d "text=这是一段要存入知识库的纯文本内容。"
```

**响应示例**: `{"kb_id":1,"source_id":"text_xxx","source_type":"text","chunks_count":1}`

---

### 3.4 删除文档

| 方法 | 路径 | 说明 |
|------|------|------|
| DELETE | `/api/knowledge-bases/{kb_id}/documents/{source_id}` | 按上传时返回的 source_id 删除该文档全部向量 |

```bash
curl -s -o /dev/null -w "%{http_code}" -X DELETE \
  "http://localhost:8000/api/knowledge-bases/1/documents/pdf_abc123def456" \
  -H "Authorization: Bearer $TOKEN"
```

**成功**: HTTP 204。

---

### 3.5 检索

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/knowledge-bases/{kb_id}/search` | 语义 / 关键词 / 混合检索 |

**请求体**:
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| query | string | 是 | 检索语句 |
| search_type | string | 否 | `semantic`（默认） / `keyword` / `hybrid` |
| top_k | int | 否 | 返回条数，1–50，默认 10 |

```bash
# 语义检索
curl -s -X POST http://localhost:8000/api/knowledge-bases/1/search \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"query":"如何配置环境","search_type":"semantic","top_k":5}'

# 关键词检索
curl -s -X POST http://localhost:8000/api/knowledge-bases/1/search \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"query":"配置 环境","search_type":"keyword","top_k":5}'

# 混合检索
curl -s -X POST http://localhost:8000/api/knowledge-bases/1/search \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"query":"配置 环境","search_type":"hybrid","top_k":5}'
```

**响应**: `{"results":[{"id":"...","score":0.92,"chunk_text":"...","metadata":{...}}]}`

## 4. 对话与工具

### 4.1 对话（含工具调用）

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/chat` | 多轮对话；可自动调用知识库检索工具 |

**请求体**:
| 字段 | 类型 | 说明 |
|------|------|------|
| kb_id | int | 检索时使用的知识库 ID |
| messages | array | 消息列表，每项 `{ "role": "user"\|"assistant"\|"system", "content": "..." }` |

```bash
curl -s -X POST http://localhost:8000/api/chat \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{
    "kb_id": 1,
    "messages": [{"role":"user","content":"根据知识库总结一下主要内容"}]
  }'
```

**响应**: `{"message":{"role":"assistant","content":"..."},"tool_calls":null}`（若模型调用了检索工具，会先执行再返回最终回复，`tool_calls` 可能非空）。

---

## 5. 错误与状态码

| 状态码 | 说明 |
|--------|------|
| 200 | 成功 |
| 204 | 成功且无响应体（如删除） |
| 400 | 请求参数错误 |
| 401 | 缺少或无效 `Authorization: Bearer <access_token>` |
| 404 | 资源不存在（如知识库/文档） |
| 503 | 依赖未配置（如 LLM/Pinecone） |

错误响应体示例: `{"detail": "错误信息"}`
